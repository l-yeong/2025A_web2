본체]
    - 메모리(RAM) : 휘발성메모리 주기억장치 이명서 실행중인 상태에서만 메모리 저장
    - SDD/HDD(ROM) : 비휘발성메모리 보조기억장치 이명서 비실행중인 상태에서 메모리 저장
    * 바둑게임 설치(15GB) 하면서 보조기억장치에 저장, 바둑게임 실행하면 필요한데이터를 주기억장치에 저장
    * 보조기억장치 용량이 크고 주기억장치는 속도 빠르다.

[ 개발 주요 저장 ] * 저장하고자 하는 데이터의 (실무/육하원칙 )사용처 생각
    1) 크롬(브라우저F12) 로컬스토리/세션저장소  : 클라이언트(유저) 환경에 저장, 보안 취약, 설정 정보
    2) 데이터베이스                         : 영구저장, 자바서버 외 DB서버 에 저장,
    3) 톰캣(Spring서버)세션                 : 자바서버 저장, 서버 과부하, 보안 안전, 주로 로그인 상태
    4) 레디스                              : 임시저장(캐싱), 주로 실시간(속도) 정보/DB조회 결과
    * 자율주행/AI, 서버가 통신이 미흡하거나 환경이 없을때는 레디스 활용

[ Redis ]
    - 정의 : 인메모리 기반의 데이터 저장소
    - 목적 : 관계형 데이터베이스가 보조기억장치 이므로 빠른 메모리 처리를 사용할때
        MYSQL/ORACLE : SQL
        REDIS : MYSQL(객체지향- GET/SET)
    - 구조 : KEY-VALUE, MAP/JSON/DTO 구조와 비슷
    - 캐싱 : Select 할때마다 데이터베이스 접근/요청하면 느리거나 성능 저하 되므로 한번 SELECT한 결과를 캐싱 한다.
    - 설치
        방법1) Spring 내장 Redis 인메모리
        방법2) 설치형 : Redis 자체 설치, https://redis.io/ko
        1. 윈도우/로컬 환경 : https://github.com/redis-windows/redis-windows/releases?utm_source=chatgpt.com
        2. 해당 버전 클릭 :
            선택1] Redis-8.2.2-Windows-x64-msys2-with-Service.zip (백그라운드/자동 실행)
            선택2] Redis-8.2.2-Windows-x64-msys2.zip              (직접 실행)
        3. 압축풀기 -> 압축 푼 폴더내 -> 'redis-server.exe'          (관리자 권한 실행)
            -> CMD창 켜지면서 가운데에 레디스 서버의 PORT 확인
            -> CMD창 직접 닫기/종료 : 서버가 꺼진 상태
            * 서버가 꺼지면 저장된 모든 정보는 사라진다.
    - 레디스 PORT를 6379 사용한다.
    - 스프링과 레디스 연동
        1. Spring Data Redis 의존성 추가 : implementation 'org.springframework.boot:spring-boot-starter-data-redis'
        2, application.properties
            - spring.data.redis.host=localhost
            - Spring.data.redis.port=6379
